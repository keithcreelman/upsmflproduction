# Parity Definition Policy

- `data_parity`: source records and derived facts match within approved tolerance.
- `workflow_parity`: user and commissioner workflows produce the same approved outcomes.
- `required_visual_parity`: pages or embeds that must remain visually stable.
- `approved_non_parity`: intentional redesign or behavior changes approved by ADR.
- `modeled_history_tolerance`: acceptable variance for normalized historical outputs.
